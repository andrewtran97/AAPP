# AAPP Governed AI Agent Schemas

## Status

`REFERENCE`

These schemas define advisory output contracts. They do not implement models, inference, training, tool execution, or final authority.

## Shared Envelope

Every advisory result must bind:

- `schema_version`
- `proposal_id`
- `agent_role`
- `source_receipt_refs`
- `scope_id`
- `tenant_id`
- `input_digest`
- `model_ref`
- `policy_ref`
- `created_at`
- `confidence`
- `uncertainties`
- `abstained`
- `proposal`

Missing source receipts, scope, tenant context, or input digest makes the proposal malformed for governed use.

AI cannot mint or modify trusted scope, capability, policy allow, approval, evidence identity, receipt identity, signature, verified digest, merge decision, or deployment decision.

## Triage Agent

### Purpose

Normalize findings, cluster likely duplicates, recommend owner and priority, and abstain when evidence is insufficient.

### Proposal Fields

- normalized finding type;
- evidence-backed cluster references;
- recommended owner;
- recommended priority;
- rationale references;
- uncertainty and abstention reason.

### Forbidden Effects

No patch, execution, suppression, policy decision, or incident closure.

## Patch Agent

### Purpose

Produce a candidate change proposal for deterministic validation and configured review.

### Proposal Fields

- candidate diff reference;
- exact file manifest;
- risk class;
- declared effects and forbidden effects;
- test plan;
- recovery class and recovery requirements;
- residual risks.

### Forbidden Effects

No direct file mutation, commit, push, merge, deployment, scope change, or capability issuance.

## Recovery Agent

### Purpose

Classify a failed or risky state change and propose typed recovery steps.

### Proposal Fields

- recovery class: `REVERSE`, `COMPENSATE`, `MITIGATE`, `REVOKE`, `CONTAIN`, or `IRREVERSIBLE`;
- pre-state reference;
- preconditions;
- ordered typed steps;
- idempotency requirements;
- verification plan;
- residual risks and escalation conditions.

### Forbidden Effects

No direct recovery execution, destructive action, evidence deletion, or incident closure.

## Explain Impact Agent

### Purpose

Explain affected components and impact dimensions using evidence references.

### Proposal Fields

- affected assets and components;
- confidentiality, integrity, availability, financial, operational, and compliance impact dimensions when applicable;
- evidence references;
- uncertainties;
- unsupported inferences explicitly marked.

### Forbidden Effects

No final risk acceptance, policy decision, approval, action order, or truth claim.

## Validation Chain

```text
AI proposal -> schema validation -> authority-reference validation
-> deterministic policy/capability gates -> tests/verifier
-> configured approval -> evidence record
```

An invalid proposal is rejected as data; it does not become trusted evidence merely because it was generated by an approved model.

## Data Separation

### TARGET

- Evidence Vault
- Incident & Analyst Vault
- Runtime Telemetry Zone
- Training Candidate Quarantine
- Approved Training Corpus
- Model & Evaluation Registry

Learning laws:

- No source receipt -> no candidate.
- No rights -> no training.
- No redaction -> no corpus admission.
- No tenant boundary -> no corpus admission.
- No dataset snapshot -> no reproducibility claim.
- No temporal holdout -> no improvement claim.
- No regression evaluation -> no promotion.
- No rollback model -> no deployment.

Online self-training from raw prompts or live evidence is `OUT_OF_SCOPE`.

## Evaluation Identity

A reproducible evaluation pins model digest, prompt/schema version, policy digest, evaluation-pack digest, fixtures, seed, environment, timestamp, and result.

## Non-Goals

- model implementation or selection;
- inference routing;
- training pipeline;
- online learning;
- agent tool authority;
- automatic patch or recovery execution;
- final security judgment.
